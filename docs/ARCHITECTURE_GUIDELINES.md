# 前端架构设计与开发规范

## 1. 配置管理规范

### 1.1 配置集中管理
- **所有配置必须通过 `config.js` 文件管理**，不得在代码中硬编码配置值
- **配置结构**：包含后端端口、API 路径前缀、WebSocket 路径等核心配置
- **动态主机名**：支持局域网访问，使用 `window.location.hostname` 构建 API 地址
- **配置验证**：内置配置有效性检查，确保配置正确加载

### 1.2 端口配置管理
- **后端端口变更**：修改 `config.js` 中的 `BACKEND_PORT` 值
- **前端端口**：通过启动命令指定，无需修改配置文件
- **配置优先级**：默认配置 → 环境配置 → 运行时配置

### 1.3 API 调用规范
- **使用 `API.getApiUrl()` 方法**构建 API 地址，确保一致性
- **WebSocket 连接**使用 `API.WS_URL`，确保连接地址正确
- **API 路径格式**：以 `/` 开头，如 `/agents/status`

## 2. 代码开发规范

### 2.1 文件结构
- **主页面**：`index.html` - 包含主要功能和布局
- **配置文件**：`config.js` - 集中管理所有配置
- **功能页面**：
  - `requests.html` - 请求统计页面
  - `handover.html` - 任务交接页面
- **文档**：
  - `DEPLOYMENT_GUIDE.md` - 部署指南
  - `ARCHITECTURE_GUIDELINES.md` - 架构设计与开发规范

### 2.2 代码风格
- **缩进**：使用 2 个空格进行缩进
- **命名规范**：
  - 变量和函数：驼峰命名法（camelCase）
  - 常量：大写蛇形命名法（UPPER_SNAKE_CASE）
- **注释**：
  - 关键功能和复杂逻辑必须添加注释
  - API 调用必须添加用途说明
  - 配置项必须添加说明

### 2.3 错误处理
- **API 调用必须包含错误处理**，使用 try-catch 捕获异常
- **错误信息必须清晰**，包含具体的错误原因
- **加载状态管理**：
  - API 调用前设置 `loading = true`
  - API 调用后无论成功失败都设置 `loading = false`
  - 错误时显示友好的错误信息

### 2.4 性能优化
- **减少 API 调用**：合理使用缓存和批量请求
- **资源加载**：
  - 外部资源（如 Vue.js）使用 CDN 并添加版本号
  - 内部资源（如 config.js）确保正确加载
- **代码组织**：
  - 相关功能模块化
  - 避免重复代码
  - 使用 computed 属性缓存计算结果

## 3. 故障排查规范

### 3.1 常见错误排查
- **数据加载失败**：
  1. 检查后端服务是否运行
  2. 检查配置文件中的端口是否正确
  3. 检查 API 路径是否正确
  4. 查看浏览器控制台的错误信息
  5. 使用 `console.log(window.LOBSTER_CONFIG)` 检查配置
- **WebSocket 连接失败**：
  1. 检查后端 WebSocket 服务是否启用
  2. 检查 `WS_URL` 是否正确
  3. 检查协议是否匹配（http 对应 ws，https 对应 wss）
- **页面显示异常**：
  1. 检查浏览器控制台的 JavaScript 错误
  2. 检查 CSS 样式冲突
  3. 检查响应式设计是否正确

### 3.2 调试工具
- **浏览器开发者工具**：
  - Console 标签页：查看 JavaScript 错误和日志
  - Network 标签页：监控 API 请求和响应
  - Elements 标签页：检查 DOM 结构和 CSS
- **配置调试**：
  - 在控制台输入 `console.log(window.LOBSTER_CONFIG)` 查看当前配置
  - 使用 `API.getApiUrl('/agents/status')` 验证 API 地址
- **网络调试**：
  - 使用 Postman 或 curl 测试 API 端点
  - 检查网络连接和防火墙设置

## 4. 部署流程规范

### 4.1 开发环境部署
1. **启动后端服务**：
   ```bash
   cd backend
   python3 main.py
   ```
2. **启动前端服务**：
   ```bash
   cd frontend
   python3 -m http.server 5173
   ```
3. **访问地址**：`http://localhost:5173`
4. **验证**：
   - 检查数据是否加载成功
   - 检查 WebSocket 是否连接
   - 测试核心功能

### 4.2 生产环境部署
1. **配置准备**：
   - 修改 `config.js` 中的 `BACKEND_PORT` 值
   - 确保所有配置项正确
2. **文件部署**：
   - 将 `frontend` 目录下的所有文件复制到 Web 服务器根目录
   - 确保 Web 服务器配置正确
3. **反向代理配置**（可选）：
   - 配置 Nginx 或 Apache 作为反向代理
   - 将 API 请求转发到后端服务
4. **验证**：
   - 检查页面加载是否正常
   - 检查 API 调用是否成功
   - 测试所有功能
