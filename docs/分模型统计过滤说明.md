# 📊 分模型统计 - 内部模型过滤说明

**实施日期**: 2026-03-27  
**版本**: v1.0.9

---

## 📋 问题背景

### 现象

前端"分模型请求统计"中显示 `delivery-mirror` 模型，用户困惑这是什么。

### 原因

`delivery-mirror` 是 **OpenClaw 的内部响应处理机制**，不是真实的大模型服务。

**特征**:
```json
{
  "api": "openai-responses",
  "provider": "openclaw",
  "model": "delivery-mirror",
  "usage": {
    "totalTokens": 0  ← 0 Token 消耗
  }
}
```

---

## 🎯 解决方案

### 方案 A: 前端过滤（已实施）✅

**优点**:
- ✅ 快速实施
- ✅ 不影响数据采集
- ✅ 用户看不到无关数据
- ✅ 保留后端数据完整性

**实施内容**:
1. 前端计算属性过滤内部模型
2. UI 显示"已过滤内部模型"提示
3. 添加说明文字

---

## 🔧 技术实现

### 1. 计算属性过滤

**index.html** - `modelStatsDisplay`:

```javascript
modelStatsDisplay() {
  if (!this.providerStats.models?.models) return []
  
  // 过滤掉 OpenClaw 内部模型
  const internalModels = ['delivery-mirror', 'response-mirror']
  return this.providerStats.models.models.filter(model => 
    !internalModels.includes(model.model)
  )
}
```

### 2. 检测是否过滤

```javascript
hasFilteredInternalModels() {
  if (!this.providerStats.models?.models) return false
  const internalModels = ['delivery-mirror', 'response-mirror']
  return this.providerStats.models.models.some(m => 
    internalModels.includes(m.model)
  )
}
```

### 3. UI 显示

**标题栏**:
```html
<h4>📊 分模型请求统计</h4>
<span v-if="hasFilteredInternalModels">
  ℹ️ 已过滤内部模型
</span>
```

**说明文字**:
```html
<div style="background: #fff8e6; border-left: 3px solid #f59e0b;">
  💡 <strong>说明</strong>：已自动过滤 OpenClaw 内部模型
  （如 delivery-mirror），这些模型用于响应格式化，
  不产生实际 Token 消耗。
</div>
```

### 特征识别

```javascript
// 内部模型特征
const internalModels = {
  'delivery-mirror': '响应交付',
  'response-mirror': '响应处理'
}

// 识别方法
function isInternalModel(modelName) {
  return modelName.endsWith('-mirror') || 
         modelName.startsWith('internal-')
}
```

## 📝 维护说明

### 添加新的内部模型

如果 OpenClaw 新增内部模型，只需更新过滤列表：

```javascript
const internalModels = [
  'delivery-mirror',
  'response-mirror',
  'new-internal-model'  // ← 添加新的
]
```

### 取消过滤（如需要）

如果需要显示内部模型，注释过滤逻辑：

```javascript
modelStatsDisplay() {
  // const internalModels = ['delivery-mirror', ...]
  // return this.providerStats.models.models.filter(...)
  return this.providerStats.models.models  // ← 不过滤
}
```
