# 龙虾办公室 - 今日任务数据为空问题修复记录

**日期**: 2026-04-01
**维护**: dev-claw (CTO)

---

## 📋 问题描述

**现象**: 龙虾办公室前端获取今日任务数据为空，与实际不符。

**预期行为**: 应显示今日应执行的定时任务（如心跳检查、每日提醒等）。

---

## 🔍 问题排查

### 1. 数据流程分析

```
OpenClaw Cron 任务
    ↓
cron_sync.py 同步到数据库 (TaskRecord)
    ↓
main.py API 查询今日任务
    ↓
前端显示
```

### 2. 排查步骤

1. **检查 API 返回**: `curl http://localhost:8000/api/v1/agents/tasks` 返回空
2. **检查数据库**: 任务记录存在，但 `created_at` 时间不正确
3. **检查过滤逻辑**: 任务被过滤规则错误过滤

---

## 🐛 根本原因

**时区转换 Bug**: `cron_sync.py` 在同步任务时使用 `datetime.now(CONFIG_TZ)` 转换为 UTC 作为 `created_at`，导致以下问题：

- 当前时间（北京）: 2026-04-01 08:xx
- 转换后 UTC: 2026-04-01 00:xx
- 存入数据库: 2026-03-31 16:xx（错误！）

API 查询今日任务范围（UTC）: `2026-03-31 16:00 ~ 2026-04-01 16:00`

由于任务的 `created_at` 是 2026-03-31 16:xx，刚好落在查询范围边界，可能被过滤或查询逻辑有问题。

---

## ✅ 修复方案

### 修复 1: 使用真正的 UTC 时间（临时方案）

**文件**: `backend/cron_sync.py`

```python
# 修改前（错误）
now_utc = to_utc(datetime.now(CONFIG_TZ))

# 修改后
now_utc = datetime.utcnow()
```

### 修复 2: 使用任务计划执行时间（根本方案）

**文件**: `backend/cron_sync.py`

```python
# 计算任务的计划执行时间（用于 created_at）
# 优先使用任务的计划执行时间，而不是当前同步时间
if 'target_datetime' in task:
    # 拆分后的任务（如每2小时），使用目标时间
    task_created_at = task['target_datetime'].astimezone(UTC).replace(tzinfo=None)
elif next_info['datetime'] is not None and next_info['is_today']:
    # 有计划执行时间且是今天
    task_created_at = next_info['datetime'].astimezone(UTC).replace(tzinfo=None)
else:
    # 没有计划时间，使用当前 UTC 时间
    task_created_at = datetime.utcnow()
```

### 修复 3: 更新任务时同时更新 created_at

**文件**: `backend/cron_sync.py`

```python
# 更新任务时
existing.created_at = task_created_at  # 使用计划执行时间
```
