## 🚀 快速开始

### 1. 启动应用

根据你选择的部署方式启动应用：

**Docker 方式**:
```bash
docker-compose up -d
```

**脚本方式**:
```bash
./start.sh
```

**源码方式**:
```bash
# 启动后端
cd backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000

# 启动前端（新终端）
cd frontend
python -m http.server 5173 --bind 0.0.0.0
```

### 2. 访问应用

打开浏览器，访问：
- **前端页面**: http://localhost:5173
- **API 文档**: http://localhost:8000/docs

### 3. 配置（首次使用）

系统会自动配置并启动，无需手动修改配置文件。

**注意**：配置文件会在启动时自动生成，用户可以通过前端界面修改同步频率等设置。

## 🎯 Agent 状态监控

### 状态分类

系统将 Agent 分为三个状态区域：

#### 🟢 空闲区

**状态判定**（按优先级）：
1. 没有正在进行的子 Agent
2. 没有正在进行的 cron 类型 task
3. 空闲时间超过配置阈值

**显示内容**：
- Agent 名称
- 空闲时间
- 最近使用模型
- 最后活动时间

**操作**：
- 点击查看详情
- 查看历史记录

#### 🟡 对话中区

**状态判定**：
- 有正在进行的子 Agent（subagent）

**显示内容**：
- Agent 名称
- 当前对话内容（摘要）
- 对话持续时间
- 最近使用模型

**操作**：
- 点击查看详情
- 查看对话内容

#### 🔴 工作中区

**状态判定**：
- 有正在进行的 task（非 cron 类型）

**显示内容**：
- Agent 名称
- 任务名称
- 任务进度
- 预计完成时间

**操作**：
- 点击查看详情
- 查看任务详情

### Agent 卡片

每个 Agent 显示为一个卡片，包含：

```
┌──────────────────┐
│ Agent 名称       │
├──────────────────┤
│ 🟢 空闲          │
│ 空闲: 15分钟     │
│ 模型: GPT-4     │
│ 最后: 10:30     │
└──────────────────┘
```

### 实时更新

- 状态通过 WebSocket 实时更新
- 无需手动刷新
- 前端数据刷新频率：可配置（默认 300 秒）
- 后端 Agent 同步频率：可配置（默认 5 分钟）

### Agent 扫描逻辑

系统会自动检索本地 OpenClaw 文件夹，找到 agents 目录，从中获取数据：

1. **优先扫描** `~/.openclaw/agents`（OpenClaw 默认路径）
2. **如果没有找到**，扫描当前工作目录的 `agents` 子目录
3. **如果还是没有**，扫描所有可能的 OpenClaw 安装路径

**识别标准**：包含 `sessions` 目录的文件夹会被识别为 Agent

**注意**：系统会自动完成 Agent 扫描和数据获取，无需用户手动配置。

| 列 | 说明 |

| 任务名称 | 任务描述 |

## 🔔 督促功能

点击底部「🔔 督促功能」进入督促功能页面。

### 手动督促

#### 发送督促消息

1. 选择要督促的 Agent
2. 输入督促消息内容
3. 点击「发送」

#### 督促消息模板

系统提供预定义模板：

| 模板 | 内容 |
|------|------|
| 通用提醒 | 请继续完成当前任务 |
| 进度询问 | 请问当前任务进度如何？ |
| 超时提醒 | 任务即将超时，请加快进度 |
| Token 提醒 | Token 使用量已接近预算 |

### 自动督促

系统根据配置自动发送督促消息。

#### 触发条件

| 条件 | 说明 | 默认阈值 |
|------|------|---------|
| 任务超时 | 任务执行超过指定时间 | 3600 秒 (1 小时) |
| 长时间无更新 | Agent 长时间无活动 | 1800 秒 (30 分钟) |
| Token 超预算 | Token 使用超过预算阈值 | 90% |
| 进度停滞 | 任务进度长时间无变化 | 3600 秒 (1 小时) |

#### 配置自动督促

在配置文件中设置：

```json
{
  "reminder": {
    "enabled": true,
    "task_timeout": 3600,
    "no_update": 1800,
    "token_threshold": 0.9,
    "progress_stagnation": 3600
  }
}
```

### 督促历史

查看所有督促记录：

- 发送时间
- 接收 Agent
- 督促类型（手动/自动）
- 督促内容
- 发送状态

## ⚙️ 配置管理

### 打开配置面板

点击顶部导航栏的「配置」按钮，或点击底部「⚙️ 配置」。

### 可修改的配置项

用户可以通过前端界面修改以下同步频率设置：

#### 前端刷新配置（秒）

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 自动刷新 | 页面自动刷新频率 | 300 |
| 状态数据刷新 | 状态数据刷新频率 | 300 |
| 统计数据刷新 | 统计数据刷新频率 | 300 |

#### 后端同步配置（分钟）

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| Agent 同步 | Agent 状态同步频率 | 5 |
| 任务同步 | 任务数据同步频率 | 10 |
| 请求同步 | 请求日志同步频率 | 15 |
| Cron 同步 | Cron 任务同步频率 | 90 |

### 保存配置

修改配置后：
1. 点击「保存」按钮
2. 配置立即生效
3. 配置会自动保存

### 前端配置

**注意**：前端配置文件 `frontend/config.js` 会在启动时自动生成，无需手动编辑。

生成的配置会自动适配当前访问地址，支持局域网访问。

### 重置配置

点击「重置」按钮可以：
- 恢复默认配置
- 清空自定义配置

## ❓ 常见问题

### Agent 状态相关

**Q: Agent 状态多久更新一次？**

A: 默认 2 分钟，可以在配置中修改同步间隔。也可以手动点击「刷新」按钮立即更新。

**Q: Agent 状态是如何判定的？**

A: 按以下优先级判定：
1. 有正在进行的 subagent → 对话中
2. 有正在进行的非 cron task → 工作中
3. 否则判定为空闲

**Q: 为什么有些 Agent 不显示？**

A: 检查：
- Agent 是否在 OpenClaw 平台中
- Agent 扫描目录配置是否正确
- 网络连接是否正常
- 日志中是否有错误

### 任务管理相关

**Q: 任务列表为什么是空的？**

A: 可能原因：
- 暂无进行中的任务
- 时间筛选条件不正确
- 数据同步未完成

**Q: 如何移交任务？**

A: 
1. 点击任务查看详情
2. 点击「移交任务」
3. 选择目标 Agent
4. 确认移交

**Q: 任务移交后能撤销吗？**

A: 目前不支持撤销，需要重新移交回来。

### 数据统计相关

**Q: 统计数据为什么不更新？**

A: 检查：
- 同步间隔配置
- 网络连接
- 后端日志

**Q: 如何查看历史统计数据？**

A: 目前只展示实时统计，历史数据功能在后续版本中推出。

### 督促功能相关

**Q: 如何关闭自动督促？**

A: 在配置中将「启用自动督促」设置为 false。

**Q: 督促消息会发送到哪里？**

A: 督促消息记录在系统中，可在督促历史查看。如果配置了飞书 Webhook，也会推送到飞书。

**Q: 可以自定义督促消息吗？**

A: 手动督促可以输入自定义内容，自动督促使用系统模板。

### 配置相关

**Q: 配置保存在哪里？**

A: 系统会自动管理配置，用户通过前端界面修改的配置会自动保存。

**Q: 修改配置后需要重启吗？**

A: 不需要，配置修改后立即生效。

**Q: 如何备份配置？**

A: 系统会自动管理配置，无需手动备份。

### 其他问题

**Q: 支持多用户吗？**

A: 当前版本是单用户设计，多用户功能在后续版本中推出。

**Q: 数据会上传到云端吗？**

A: 不会，所有数据存储在本地数据库中。

**Q: 如何备份数据？**

A: 复制 `backend/openclaw_office.db` 文件即可备份数据库。

**Q: 如何恢复数据？**

A: 将备份的 `openclaw_office.db` 复制到 `backend/` 目录，重启服务。
